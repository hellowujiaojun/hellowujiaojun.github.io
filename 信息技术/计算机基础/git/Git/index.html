工作区、暂存区和版本库的概念
版本库

在初始化或克隆一个Git仓库时，会创建一个名为.git的隐藏目录，该目录就是版本库。

版本库中存储了很多配置信息，日志信息和文件版本等。

工作区

GIt仓库目录就是工作区。工作区中存放开发的代码。

暂存区

暂存区Stage位于.git目录下的index文件中。

暂存区是一个临时保存修改文件的地方。

GIt工作目录下文件的两个状态
Untracked

untracked未跟踪（未被纳入版本控制）。

Tracked

tracked已跟踪（被纳入版本控制）。

tracked状态下还有三种状态：

  1. Unmodified未被修改状态。
  2. Modified已修改状态。
  3. Staged已暂存状态。
本地仓库的操作
查看文件状态

在Git仓库目录下使用git status指令查看文件状态。

使用git status -s可以使输出信息更简洁。

将文件加入暂存区

使用git add 将untracked文件加入，变成tracked文件，默认进入暂存区。

使用git reset命令可以将暂存文件取消跟踪。

提交到本地仓库

git commit 命令可以将暂存区里的文件提交给本地仓库。

若不加-m参数，会默认打开本地编辑器，编写相关日志信息。

需要注意，只有在暂存区的文件才会被提交，因此被修改的文件README.md未被提交。需要使用add指令将该文件加入暂存区后才能被提交。

删除文件

使用git rm指令可以删除工作区文件，删除后会自动加入暂存区。

该操作删除只是删除了工作区的文件，还需要提交才能删除本地仓库的文件。

非常不推荐使用其他方式删除工作区文件，会造成一些小麻烦。

虽然识别出已经删除，但是并没能在暂存区中，无法提交给仓库。此时还需要将该文件（已经不存在）添加到暂存区后再提交一次。

远程仓库操作
查看远程仓库
添加远程仓库

运行git remote add <shortname> <url> 添加一个新的远程Git仓库，同时指定一个可以引用的简写。

一个本地仓库可以对应多个远程仓库，但<shortname>需要不同。

从远程仓库克隆

使用git clone可以克隆远程仓库，默认会将远程仓库每一个文件的每一个版本都被克隆。

移除无效的远程仓库

使用git remote rm可以移除本地和远程仓库的联系。

从远程仓库抓取与拉取

git fetch是从远程仓库获取最新版本到本地仓库，不会自动merge

会以二进制文件存放在.git目录下object目录。

需要手动合并才能merge

git pull是从远程仓库获取最新版本并merge到本地仓库。

如果当前本地仓库不是从远程克隆的，而是本地创建的仓库且有文件，此时再从远程仓库拉取会报错（fatal: refusing to merge unrelated histories)，解决此问题可以在git pull命令后加入参数–allow-unrelated-histories。

输入后会自动拉取并merge且需要输入日志。

推送到远程仓库

将本地代码推送至远程仓库使用git push命令

使用时会要求输入远程仓库的账户密码。
